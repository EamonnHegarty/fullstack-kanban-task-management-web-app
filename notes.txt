A user has board, boards have tasks, tasks have subtasks 

one to many, one to many, one to many 

so board tables will have UserID as the foreignKey, task tables will have boardID as their forign key 

Boards:

GET /board: Retrieve all boards.
GET /board/{board_id}: Retrieve a specific board.
POST /board: Create a new board.
PUT /board/{board_id}: Update an existing board.
DELETE /board/{board_id}: Delete a board.



Tasks:

GET /board/{board_id}/task: Retrieve all tasks for a specific board.
GET /board/{board_id}/task/{task_id}: Retrieve a specific task within a board.
POST /board/{board_id}/task: Create a new task within a board.
PUT /board/{board_id}/task/{task_id}: Update an existing task within a board.
DELETE /board/{board_id}/task/{task_id}: Delete a task within a board.



Subtasks:

GET /board/{board_id}/task/{task_id}/subtask: Retrieve all subtasks for a specific task.
GET /board/{board_id}/task/{task_id}/subtask/{subtask_id}: Retrieve a specific subtask within a task.
POST /board/{board_id}/task/{task_id}/subtask: Create a new subtask within a task.
PUT /board/{board_id}/task/{task_id}/subtask/{subtask_id}: Update an existing subtask within a task.
DELETE /board/{board_id}/task/{task_id}/subtask/{subtask_id}: Delete a subtask within a task.
